REPLIT AGENT PROMPT — Paramodus Feature Expansion (NanoClaw-Inspired)
You are working inside a Python project called Paramodus — a desktop-native AI assistant built with Agno, pywebview, LanceDB (local RAG), FastEmbed, and SQLite. The stack is pure Python + HTML/CSS/JS. The repo structure is:


app.py — entry point, pywebview window


agents/workspace_agent.py — Agno agent logic, multi-LLM switching


api/bridge.py — Python↔JS bridge


database.py — SQLite chat history


ui/ — frontend assets


Constraint: This runs on Replit Free Tier. That means:


No Docker, no containers, no background daemons


No persistent filesystem outside the Repl directory


Keep RAM usage minimal — no heavy embedding batch loads at startup


No npm/Node.js unless absolutely necessary — stay Python-native


Use SQLite for all persistence (already in place)


Your task: Implement the following features inspired by the NanoClaw architecture, adapted for the Paramodus Python/Agno stack. Do NOT try to replicate Docker isolation — simulate group/session isolation in software using SQLite scoping and per-session agent instances.
Features to implement (in priority order):


Isolated Group Contexts — Add a groups table to SQLite. Each group has its own system_prompt / memory string (equivalent to CLAUDE.md). When the user selects a group in the UI, the Agno agent is re-instantiated with that group's memory loaded as the system prompt. Store and reload group memory per session.


Scheduled Tasks (lightweight) — Add a simple polling loop using Python's schedule library (or APScheduler) that runs in a background thread. Tasks are stored in SQLite with a cron expression, a prompt, and a target output (e.g., show in UI chat, log to file). On Replit free tier, use threading — NOT subprocess or Docker.


Skill-style extensibility — Create a skills/ folder with individual Python modules (e.g., skills/web_search.py, skills/summarize_pdf.py). Each skill exposes a register(agent) function that adds a tool to the Agno agent. In workspace_agent.py, dynamically import and register all skills found in the skills/ folder at startup.


Agent Swarms (minimal version) — Using Agno's team/multi-agent support, allow the user to select a "team mode" in the UI dropdown. In team mode, spin up a Researcher agent + a Writer agent that collaborate. Pass the combined result back to the frontend. Keep both agents lightweight (use Groq Llama 3 or Gemini Flash for cost/speed).


Per-session message queues — Add a simple in-memory asyncio.Queue or queue.Queue per active group/session. Incoming messages from the JS frontend are queued and processed sequentially per group to prevent race conditions.


Additional instructions:


Keep all new dependencies minimal; prefer stdlib where possible


Update requirements.txt with any new pip packages added


Do not break the existing pywebview window or LanceDB RAG functionality


Add a SKILLS_README.md explaining how to write and register a new skill


Use type hints throughout new code


Test that python app.py still launches correctly after changes
